  Code（可以一次给完，它会自己分步做；如果你想控制节奏，就按「阶段」分四次发）：

  请帮我构建一个本地 Web 版 PowerShell 会话管理器，用于按项目分类管理多个
  claude/codex 终端会话。请先给出实现计划再动手。

  ## 背景
  我经常同时开很多个 PowerShell 窗口跑 claude code / codex，找窗口很麻烦。
  我想用浏览器页面按「项目 → 任务」两级分类来管理这些终端会话。

  ## 技术栈（已定，不要更换）
  - 后端：Node.js + Express + WebSocket(ws) + node-pty
    - node-pty 在 Windows 上编译麻烦的话，改用带预编译二进制的
      @homebridge/node-pty-prebuilt-multiarch
  - 前端：原生 HTML/CSS/JS + xterm.js（含 fit addon），不要用 React/Vue 等框架
  - 数据持久化：本地 JSON 文件（data/projects.json），不要数据库
  - 运行环境：Windows 11，服务器监听 0.0.0.0 以便局域网内安卓/iPad/其他电脑
    的浏览器访问；纯局域网使用，不需要账号和 HTTPS
  - 浏览器兼容：以 Firefox 为基准，同时兼容 Chrome/Safari/移动端浏览器，
    不使用仅 Chromium 支持的 API

  ## 数据模型
  Project（项目）
    └─ Task（任务）：{ 名称, 工作目录, 启动类型: claude | codex | 纯powershell }
  每个 Task 运行时对应一个 pty 会话（同一时刻一个 task 最多一个活动会话）。

  ## 功能需求
  1. 左侧边栏：项目分类树
     - 新建项目 / 选择已有项目 / 重命名 / 删除
     - 项目下新建任务：填任务名、工作目录（文本框手动输入路径）、
       启动类型（claude / codex / 纯 powershell）
     - 每个任务显示运行状态徽标：运行中（绿）/ 已退出（灰）
  2. 点击任务 → 主区域显示该任务的终端（xterm.js）
     - 启动逻辑：node-pty 在该任务的工作目录 spawn powershell.exe；
       若类型是 claude 或 codex，pty 建立后自动向 stdin 写入对应命令回车
     - WebSocket 双向：pty 输出实时推到前端渲染；前端键盘输入实时写回 pty
     - 终端尺寸自适应容器（fit addon），resize 时同步 pty 的行列数
  3. 会话生命周期
     - 会话由服务器持有，与浏览器页面解耦：关闭/刷新网页后会话继续运行，
       重新打开页面点击任务可重连并恢复内容
     - 服务器为每个会话保留至少 5000 行滚动回放缓冲，重连时先回放缓冲
     - 每个任务面板提供「终止进程」按钮：kill 整个 pty 进程树并更新状态
     - 支持多个任务的会话同时运行，侧边栏随时切换查看
  4. Ctrl+G 大段输入
     - 终端聚焦时按 Ctrl+G：拦截该按键，弹出模态框，内含大号多行文本框
     - 点「发送」把文本原样写入 pty stdin（不自动追加回车，末尾是否回车
       由文本内容决定），关闭模态框；点「取消」直接关闭
     - Esc 关闭模态框
  5. 移动端适配
     - 窄屏时侧边栏可折叠成抽屉
     - 触屏设备上提供一个按钮等价于 Ctrl+G（因为手机没有该快捷键）

  ## 实现要求
  - 目录结构清晰：server/（后端）、public/（前端静态文件）、data/（JSON）
  - WebSocket 消息设计成 JSON 协议：{type: "output"|"input"|"resize"|"exit"|...}
  - 服务器启动后在控制台打印本机局域网访问地址
  - package.json 提供 npm start；写一个 README 说明安装、启动、局域网访问方法
  - 分阶段实现并在每阶段结束时可运行验证：
    阶段1 服务器+项目任务CRUD+侧边栏UI → 阶段2 pty会话+xterm终端流 →
    阶段3 Ctrl+G、终止进程、重连回放 → 阶段4 移动端适配与打磨
  - 每个阶段完成后停下来告诉我如何验证，等我确认再继续下一阶段

  ## 明确不做
  - 不做用户登录、不做 HTTPS、不连接互联网
  - 不做「同时弹出真实 PowerShell 窗口并镜像」：网页终端本身就是窗口

  ---
  几点补充说明：

  - node-pty 是唯一有安装风险的依赖（Windows 上原版需要 Visual Studio
  编译工具链），提示词里已让它优先用预编译版本替代，一般能免去装编译环境。
  - 安卓/iPad 访问：确保 Windows 防火墙放行该端口（首次 npm start 时 Windows
  会弹防火墙提示，勾选专用网络允许即可），然后手机浏览器访问 http://你电脑的局域网IP:端口。
  - claude/codex 在 pty 里的表现和真实窗口一致（它们本来就是终端 TUI 程序），xterm.js 能完整渲染颜色和交互界面。
  - 如果之后你想加「一个任务开多个并行会话」或「会话历史归档」，都可以在这个架构上直接扩展，先不用写进第一版。
